# BNO080_HAND_DIY 接口通讯协议

固件版本：v2.22  
硬件：自研 ESP32-S3 PCB + 2× TCA9548A + 最多 16× BNO080/BNO085

---

## 1. 硬件概览

| 项目 | 参数 |
|------|------|
| MCU | ESP32-S3（FreeRTOS 双核） |
| 串口波特率 | **2,000,000 baud**（USB CDC） |
| BLE 协议 | NimBLE-Arduino |
| 传感器 | BNO080/BNO085，I2C 地址 0x4B（ADDR=VCC） |
| Bus A | GPIO8(SDA) / GPIO9(SCL) → TCA1(0x77) → CH0-7，Core 1 独占 |
| Bus B | GPIO13(SDA) / GPIO14(SCL) → TCA2(0x70) → CH8-15，Core 0 独占 |
| 通道数量 | 16 槽（CH7 永久禁用，实际最多 15 路在线） |
| 采样率 | 100 Hz（REPORT_INTERVAL_MS = 10ms） |

## 2. BLE GATT 服务

服务 UUID：`0000FFE0-0000-1000-8000-00805F9B34FB`

| 特征 | UUID | 属性 | 说明 |
|------|------|------|------|
| DATA | `0000FFE1-...` | NOTIFY | 二进制数据帧（91B/帧，100Hz） |
| CMD  | `0000FFE2-...` | WRITE_NO_RSP | 指令下发（同串口命令字符集） |
| INFO | `0000FFE3-...` | READ | 状态摘要，每 5 s 更新一次 |

**设备广播名**：`BNO_HAND_<SN>`（默认 SN = `001`，可通过 `N` 命令修改）  
**MTU**：512 字节  
**连接间隔请求**：6-24（7.5-30 ms），Windows 通常协商至 CI = 16（20 ms）→ ~100 fps

**INFO 特征格式**（ASCII 字符串）：
```
fps=153.2 CI=16/20.0ms notify=12345 poll=180us
```

## 3. 二进制帧协议（Binary Frame，91 字节）

### 3.1 帧结构

| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 1 | uint8 | 同步头 `0xAA` |
| 1 | 1 | uint8 | 同步头 `0x55` |
| 2-5 | 4 | uint32 LE | 帧序号 `seq`（每帧 +1，从 0 起） |
| 6-9 | 4 | uint32 LE | 时间戳 `ts_ms`（毫秒，`millis()`） |
| 10-89 | 80 | — | 16 通道数据（CH0-CH15，每通道 5B） |
| 90 | 1 | uint8 | XOR 校验（字节 [2..89] 的异或） |

**总长度：91 字节**

### 3.2 通道数据格式（每通道 5 字节）

| 偏移（通道内） | 类型 | 说明 |
|------|------|------|
| 0 | int8 | qw × 127（四元数 W 分量） |
| 1 | int8 | qx × 127（四元数 X 分量） |
| 2 | int8 | qy × 127（四元数 Y 分量） |
| 3 | int8 | qz × 127（四元数 Z 分量） |
| 4 | uint8 | acc（标定精度，0-3） |

**还原公式**：`q = (int8_value) / 127.0f`，精度 ≈ 1/127 ≈ 0.0079（约 0.5°）

> CH7 永久禁用，qw/qx/qy/qz/acc 恒为 0。

### 3.3 校验计算

```python
xor = 0
for b in frame[2:90]:   # bytes [2..89]
    xor ^= b
assert frame[90] == xor
```

### 3.4 Python 解析示例

```python
import struct

FRAME_LEN = 91
SYNC = b'\xAA\x55'

def parse_frame(data: bytes) -> dict | None:
    if len(data) < FRAME_LEN or data[0:2] != SYNC:
        return None
    xor = 0
    for b in data[2:90]:
        xor ^= b
    if xor != data[90]:
        return None                          # 校验失败
    seq, ts = struct.unpack_from('<II', data, 2)
    channels = []
    for ch in range(16):
        off = 10 + ch * 5
        qw, qx, qy, qz = struct.unpack_from('4b', data, off)
        acc = data[off + 4]
        channels.append({
            'qw': qw / 127.0,
            'qx': qx / 127.0,
            'qy': qy / 127.0,
            'qz': qz / 127.0,
            'acc': acc,
        })
    return {'seq': seq, 'ts_ms': ts, 'channels': channels}
```

## 4. ASCII 帧协议（调试模式，串口专用）

通过 `A` 命令切换至 ASCII 模式。每帧一行，CSV 格式：

```
<seq>,<ts_ms>,<qw0>,<qx0>,<qy0>,<qz0>,<acc0×100>,<qw1>,...,<acc15×100>\r\n
```

| 字段 | 说明 |
|------|------|
| seq | 帧序号（十进制） |
| ts_ms | 时间戳 ms |
| qwN..qzN | 四元数分量 × 127，整数（-127 ~ 127） |
| accN×100 | 精度值 × 100（0 / 100 / 200 / 300） |

共 2 + 16×5 = 82 个字段，以 `\r\n` 结尾。

## 5. 串口 / BLE CMD 指令集

串口：发送字符 + 回车（`\r` 或 `\n`）  
BLE：写入 CMD 特征（`0xFFE2`），无需回车

| 命令 | 说明 |
|------|------|
| `R` | 软复位：挂起 Bus B 任务，重初始化所有 BNO080 |
| `Z` | Yaw 归零（提示：在上位机点击"中立位"按钮） |
| `S` | 保存 DCD 校准数据到 BNO080 Flash |
| `C` | 查询所有通道当前 acc 精度值 |
| `F` / `M` | 循环切换报告类型（见第 6 节） |
| `H` | 循环切换 I2C 频率（400→600→800→1000 kHz） |
| `A` | 切换 ASCII / Binary 串口输出模式 |
| `B` | 打印 BLE 状态（CI、notify fps、丢帧数） |
| `O` | 打印 OTA / WiFi 当前状态 |
| `G` | 手动触发 GitHub OTA 版本检查 |
| `0` | 滤波预设 0：OFF（原始值，α=1.00） |
| `1` | 滤波预设 1：fixed（α_move=0.30，α_still=0.30） |
| `2` | 滤波预设 2：gesture ★默认（α_move=0.45，α_still=0.85） |
| `3` | 滤波预设 3：smooth（α_move=0.15，α_still=0.60） |
| `N` | 单独发送：打印当前序列号 SN |
| `N<SN>` | 设置序列号（最长 8 字符，重启生效），例：`N002` |

| 0 | ARVR_RV | 0x28 | 9-DOF，AR/VR 稳定旋转向量，Yaw 绝对（需磁力计校准）**默认** |

## 7. LED 状态指示

RGB LED 共阳（LOW = 亮）：IO21=红，IO48=绿，IO47=蓝

| 颜色 / 闪烁 | 含义 |
|------------|------|
| 蓝 0.5Hz | BLE 广播中（未连接） |
| 红 4Hz 快闪 | 无传感器在线 |
| 青 1Hz | 传感器在线，未完成磁力计校准（acc < 2） |
| 绿 1Hz | 已校准，电池良好（> 3800 mV） |
| 黄 1Hz | 已校准，电池中等（3500-3800 mV） |
| 红双闪 1.3s 周期 | 低电（< 3500 mV） |
| 白 2Hz | **校准模式**（长按 GPIO12 ≥ 3 s 进入；acc ≥ 2 自动保存；短按退出） |
| 紫 2Hz | OTA 检查中（连接 WiFi / 拉取 version.txt） |
| 7 色彩虹 200ms/色 | OTA 下载 + 烧录中（完成后自动重启） |

## 9. 校准操作

| 步骤 | 操作 |
|------|------|
| 进入校准 | 长按 GPIO12（PIN_BTN）≥ 3 s，LED 变白 2Hz |
| 校准中 | 手持设备做 8 字形运动，直到 acc ≥ 2 |
| 自动保存 | acc ≥ 2 时自动调用 `saveDcdToFlash()`，LED 退出白闪 |
| 手动保存 | 串口发送 `S` 命令 |
| 取消校准 | 校准模式下短按按键（进入后至少 3 s 才可取消） |

> 磁场干扰环境（如机械臂旁）：拆下 IMU 移至空旷处校准，完成后（acc ≥ 2）发 `S` 冻结保存，再装回。
